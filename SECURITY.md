# Security Policy

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability. Use
[GitHub private vulnerability reporting](https://github.com/fishfillet-krm/maintainerkit/security/advisories/new).

Please include affected versions, reproduction details, impact, and any suggested mitigation. Do
not include credentials or private repository content.

The maintainer will acknowledge reports on a best-effort basis. No response or remediation SLA is
currently guaranteed.

MaintainerKit treats authentication, authorization, security, billing, deployment, and release behavior as protected areas. Changes in these areas require explicit human review and approval.

The MVP does not transmit repository or issue data to external services.
